<h2>Architecture Changes</h2>

<p>

The baseline model was intentionally simple. Rather than making many unrelated

changes simultaneously, modifications were introduced incrementally so that each

experiment measured the effect of a single architectural idea.

</p>

<ul>

<li>Replaced the standard GELU feed-forward block with <b>SwiGLU</b>.</li>

<li>Removed learned positional embeddings and introduced <b>Rotary Position Embeddings (RoPE)</b>.</li>

<li>Increased model width while remaining comfortably below the 2 million parameter cap.</li>

<li>Kept the original optimizer and initialization after experiments showed alternative choices were consistently worse.</li>

</ul>

<h2>Why the Final Model Was Chosen</h2>

<p>

The selected model achieved the best validation bits-per-byte (BPB) observed

during experimentation while satisfying every assignment constraint. Rather than

including every modern Transformer modification, only the changes that produced

measurable improvements were retained.

</p>

<p>

The final architecture combines three successful ideas:

</p>

<ul>

<li>SwiGLU feed-forward layers for improved representational capacity.</li>

<li>Rotary Position Embeddings for stronger positional modelling.</li>

<li>A wider hidden dimension to better utilize the available parameter budget.</li>

</ul>

<p>

Several other modern techniques (alternative initialization schemes, RMSNorm,

QK-Norm, weight tying, optimizer modifications, and output calibration) were

evaluated but discarded because they either failed to improve BPB or slightly

reduced performance under the assignment's fixed 2000-step training budget.

</p>
